Design Specifications

Placement

  • • Inside the Filter bar header row, right-aligned
  • • Desktop: 224px (w-56) fixed width, inline with "Filters" label
  • • Mobile: Full width below "Filters" label, stacks vertically
  • • Part of the filter system (counts toward active filter badge)

Search Behavior

  • • Client-side, real-time as-you-type filtering
  • • Searches: shortId, title, description, epicName, tags
  • • Case-insensitive substring match
  • • Works with existing agent/priority filters (AND logic)
  • • Column story counts update to reflect matches

Visual Feedback

  • • Matching cards: blue ring + border highlight
  • • Non-matching: dimmed to 30% opacity (still visible)
  • • Match text: yellow highlight (<mark> tag)
  • • Match count badge: "N matches" in blue pill
  • • No matches: empty state with search icon + helpful text

Input Styling

  • • Follows design-brief form input pattern
  • • Search icon (magnifying glass) left-aligned
  • • Clear button (X) appears when text is entered
  • • Keyboard shortcut hint (Cmd+K) shown when empty
  • • Focus: ring-2 ring-blue-200 border-blue-500

Keyboard Support

  • • Cmd+K / Ctrl+K: Focus search input from anywhere
  • • Escape: Clear search and blur input
  • • Debounced: 150ms delay to avoid jank during typing

State Management

  • • New searchQuery field in boardStore (Zustand)
  • • Cleared by "Clear All" filters button
  • • Persisted in store (survives page navigation)
  • • New filterBySearch() utility in sorting.ts
